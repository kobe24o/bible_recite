# 飞书云端背诵计划与跨书卷计划设计

- 日期：2026-07-17
- 状态：待用户复核

## 1. 目标

本轮为 App 增加两个内置预置计划，并建立一套免费、可复制的飞书多维表格发布协议：

1. 将《圣经经典篇章》中的 20 个经文范围作为一个预置计划。
2. 将《每卷书钥节》中的 66 个经文范围作为一个预置计划。
3. 经文范围精确保存起止章、起止节；多节范围不得退化为整章。
4. 一个计划可包含多个书卷。
5. 每个计划创建时可选择简体中文、繁体中文或英文译本。
6. 云端计划同步后，用户可修改译本、开始日期和结束日期，但不可修改经文范围。
7. App 内置官方默认发布源，并允许用户添加多个团队自建发布源。
8. 团队只需复制统一的飞书多维表格模板、维护数据并公开一个只读视图 URL，不需要付费服务。

## 2. 已确认的源数据

### 2.1 圣经经典篇章

该计划包含 20 个有序经文范围。既有完整章节，也有章内局部范围，例如路加福音 1:46-56、诗篇 139:1-12、马太福音 6:25-34。App 只采用章节和经节范围，不导入文档中的参考经文正文。

### 2.2 每卷书钥节

该计划包含新教 66 卷每卷一个钥节范围。部分条目包含多节，例如民数记 14:22-23、申命记 6:4-5、耶利米哀歌 3:22-23、以弗所书 2:8-9、帖撒罗尼迦前书 5:16-18。每个条目必须保留准确起止节。

## 3. 总体架构

飞书模板内部使用三张规范化数据表：`章节目录`、`背诵计划`、`计划经文`。发布者维护规范化表格；手机只读取 `计划经文` 的一个独立分享视图。该视图通过关联、查找引用和公式形成扁平的公开协议，因此每个发布源仍只需配置一个 URL。

App 将云端计划模板和用户正在执行的计划分开保存：

- 云端模板是从发布源同步得到的只读经文清单。
- 用户点击模板创建计划时，生成本地快照。
- 本地快照允许修改译本、开始日期和结束日期。
- 本地快照中的经卷、章节、经节和顺序不可修改。
- 已开始执行的本地计划不被后续云端修订覆盖。

## 4. 飞书多维表格设计

### 4.1 数据表：章节目录

该表预置新教 66 卷全部有效章节，每章一行。首列作为可搜索的关联下拉选项。

| 字段 | 类型 | 规则 |
|---|---|---|
| 章节键 | 文本主键 | `PSA.023｜诗篇 23`，全表唯一 |
| 经卷 OSIS | 单选 | 固定 66 个 OSIS 标识 |
| 简体卷名 | 文本 | 用于简体界面 |
| 繁体卷名 | 文本 | 用于繁体界面 |
| 英文卷名 | 文本 | 用于英文界面 |
| 章号 | 数字 | 正整数，不超过该卷章数 |
| 简体最大节数 | 数字 | 简体经文包中的有效最大节号 |
| 繁体最大节数 | 数字 | 繁体经文包中的有效最大节号 |
| 英文最大节数 | 数字 | 英文经文包中的有效最大节号 |
| 发布最大节数 | 公式 | 取三个译本最大节数的最小值，保证发布范围可用于三个译本 |
| 正典顺序 | 数字 | 用于范围先后校验 |

章节键和三个译本的最大节数由模板生成脚本从 App 的本地经文包预填，发布者不需要手工录入。选择章节时使用关联字段的搜索下拉，因此不存在无效书卷或无效章节选项。发布范围先以三个译本共同可用的节号校验；App 创建具体译本计划时，再通过现有版本映射和本地经文包做最终校验。

### 4.2 数据表：背诵计划

| 字段 | 类型 | 规则 |
|---|---|---|
| 计划 ID | 文本主键 | 发布源内永久不变；仅使用小写字母、数字和连字符 |
| 计划名称 | 文本 | 必填 |
| 计划简介 | 文本 | 可选 |
| 是否推送 | 单选 | 仅允许 `是`、`否` |
| 修订号 | 数字 | 从 1 开始；发布内容变更时递增 |
| 默认译本 | 单选 | 简体、繁体、英文 |
| 默认开始日期 | 日期 | 可选；同步后用户可改 |
| 默认结束日期 | 日期 | 可选；不得早于开始日期 |
| 来源名称 | 文本 | 如“背诵助手官方”或团队名称 |
| 协议版本 | 单选 | 当前固定为 `1` |
| 标签 | 多选 | 可选 |

取消推送或删除云端记录只影响尚未同步的设备。手机上已经同步的模板和已经创建的计划不得被远程删除。

### 4.3 数据表：计划经文

| 字段 | 类型 | 规则 |
|---|---|---|
| 条目 ID | 自动编号主键 | 飞书内部标识 |
| 所属计划 | 关联记录 | 关联 `背诵计划`，必填且只关联一个计划 |
| 经文顺序 | 数字 | 同一计划内从 1 连续递增且唯一 |
| 起始章节 | 关联记录 | 关联 `章节目录`，下拉选择且只选一项 |
| 起始节 | 数字 | 大于等于 1，不超过起始章节最大节数 |
| 终止章节 | 关联记录 | 关联 `章节目录`，下拉选择且只选一项 |
| 终止节 | 数字 | 大于等于 1，不超过终止章节最大节数 |
| 起始经卷 | 查找引用 | 从起始章节读取 OSIS，只读 |
| 终止经卷 | 查找引用 | 从终止章节读取 OSIS，只读 |
| 起始章号 | 查找引用 | 只读 |
| 终止章号 | 查找引用 | 只读 |
| 起始最大节数 | 查找引用 | 读取起始章节的发布最大节数，只读 |
| 终止最大节数 | 查找引用 | 读取终止章节的发布最大节数，只读 |
| 范围校验 | 公式 | 输出 `通过` 或明确错误原因 |
| 推送状态 | 查找引用 | 从所属计划读取“是否推送” |
| 计划 ID | 查找引用 | 公开协议字段 |
| 计划名称 | 查找引用 | 公开协议字段 |
| 修订号 | 查找引用 | 公开协议字段 |
| 默认译本 | 查找引用 | 公开协议字段 |
| 默认开始日期 | 查找引用 | 公开协议字段 |
| 默认结束日期 | 查找引用 | 公开协议字段 |
| 来源名称 | 查找引用 | 公开协议字段 |
| 协议版本 | 查找引用 | 公开协议字段 |

`范围校验` 必须检查：

1. 起止章节均已选择。
2. 起止节均为正整数。
3. 起始节不超过起始章节最大节数。
4. 终止节不超过终止章节最大节数。
5. 单条经文范围不能跨书卷；跨书卷计划通过多条记录组成。
6. 起始位置不得晚于终止位置。

### 4.4 独立分享视图

在 `计划经文` 中创建 `App公开发布` 视图：

- 筛选条件：推送状态为“是”，范围校验为“通过”。
- 排序：计划 ID 升序、经文顺序升序。
- 只展示手机同步需要的字段，不展示内部关联和辅助字段。
- 开启独立分享，并设置为互联网上获得链接的人可阅读。
- App 配置的发布源 URL 就是该视图的公开只读链接。

公开字段固定为：协议版本、来源名称、计划 ID、计划名称、修订号、默认译本、默认开始日期、默认结束日期、经文顺序、起始经卷、起始章号、起始节、终止经卷、终止章号、终止节。

## 5. App 领域模型与数据库迁移

当前 `memorization_plan` 把 `translation_id`、`book_id` 和章范围保存在计划级，`plan_task` 不保存书卷，无法支持跨书卷计划。迁移后：

- `memorization_plan` 保留用户选择的 `translation_id`、日期和进度，不再用单一 `book_id` 表达计划内容。
- `plan_task` 或新的计划内容表保存每项任务对应的 `book_id`、起止章和起止节。
- 新增云端发布源表，保存 URL、来源显示名、启用状态、最后同步时间和最后错误。
- 新增云端模板表和模板经文表，保存来源指纹、计划 ID、修订号、只读范围和同步状态。
- 云端模板唯一键为 `source_fingerprint + plan_id`。
- 用户从云端模板创建计划时复制为本地快照，并保存来源标记。

数据库迁移必须保留现有本地计划、任务、统计、成就和艾宾浩斯复习数据。

## 6. 同步规则

1. App 内置官方默认发布源 URL；用户可停用，但不需要首次配置。
2. 用户可添加多个团队公开 URL；添加时立即验证协议和至少一个完整计划。
3. 发布源指纹由规范化公开 URL 的 SHA-256 生成。
4. 每次同步只接受协议版本受支持、推送状态有效且全部范围通过本地校验的完整计划。
5. 同一计划缺行、顺序重复、元数据冲突或经文越界时，整项计划拒绝更新，保留上一次成功版本。
6. 修订号增加时更新云端模板；已经创建的执行计划不更新。
7. 云端计划消失、改为不推送或发布源暂时不可访问时，本机已有模板和执行计划不删除。
8. 网络或解析失败时显示来源级错误和上次成功同步时间，继续使用本地缓存。
9. 不在 App 中保存飞书账号、Cookie、应用密钥或访问令牌。

## 7. 用户界面

### 7.1 计划模板

- 官方默认源显示“官方云端”徽标。
- 团队源显示“云端 · 来源名称”徽标。
- 本地自建计划不显示云端徽标。
- 云端模板详情显示锁图标和“章节范围由发布者维护”。
- 创建云端计划时允许修改译本、开始日期和结束日期，章节范围只读。

### 7.2 发布源管理

在计划页或设置页增加“计划发布源”：

- 显示默认官方源、用户添加的团队源、启用状态、最后同步时间和错误状态。
- 支持粘贴公开 URL、校验、添加、停用和手动同步。
- 删除发布源只移除后续同步配置，不删除已同步模板和已创建计划。

## 8. 安全与可靠性

- 公开视图内容视为不可信输入；只解析固定字段，不执行任何内容。
- URL 仅允许 HTTPS，并限制为受支持的飞书分享域名。
- 对响应大小、记录数、文本长度和同步超时设置上限。
- 所有章节和节数都以 App 本地 66 卷目录及所选译本再次校验。
- 飞书公开页面结构变化时，同步适配器失败关闭，保留上次成功缓存，不清空本地数据。
- 公开视图不得包含个人信息、密钥、内部备注或参考经文正文。

## 9. 测试与验收

### 9.1 数据测试

- 验证两个来源文件分别解析为 20 和 66 个有序经文范围。
- 验证所有多节范围保留准确起止节。
- 验证章节目录无重复、覆盖 66 卷全部有效章节，最大节数与本地经文包一致。
- 验证无效章节、零节、越界节、反向范围和跨书卷单条范围均被拒绝。

### 9.2 同步测试

- 首次同步、重复同步和修订号升级保持幂等。
- 云端改为不推送、删除记录、URL 失效和网络中断均不删除本地计划。
- 两个团队使用相同计划 ID 时因来源指纹不同而互不覆盖。
- 非法计划更新不覆盖上一次成功版本。

### 9.3 UI 测试

- 官方、团队云端和本地计划具有明确视觉差异。
- 云端计划可修改译本和日期，不能修改章节范围。
- 简体、繁体和英文均可基于同一范围创建并执行计划。

### 9.4 完成标准

1. 飞书模板包含三张表和一个可公开读取的 `App公开发布` 视图。
2. 模板预填章节目录、20 项经典篇章计划和 66 项每卷书钥节计划。
3. App 默认配置官方发布 URL，并允许添加多个团队源。
4. 跨书卷计划可以创建、同步、执行和持久化。
5. 已同步计划不会因云端撤回或错误而从手机删除。
