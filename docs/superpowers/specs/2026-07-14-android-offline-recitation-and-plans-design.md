# Android 离线背诵、中文目录与计划设计

- 日期：2026-07-14
- 状态：设计已确认，等待实施计划
- 目标平台：本轮 Android 优先，接口保持跨平台可移植

## 1. 目标

本轮把现有离线经文浏览基础扩展为可实际背诵的 Android APK：中文界面显示中文经卷名；用户能从经文页开始逐节或连续背诵；普通话和英语均由本机模型实时识别；应用解释遗漏、替换、乱序和待确认内容；用户能从预置章节或自选范围创建 1–365 天计划，并在今日页和统计页查看本地进度。

所有经文、模型、识别运行库、计划和背诵记录均在设备本地工作。飞行模式或完全断网时，浏览、识别、判定、计划和统计仍可使用。

## 2. 范围

### 2.1 本轮包含

- 简体中文、繁体中文和英文经卷名称映射，内部继续使用稳定 OSIS ID。
- Android arm64 离线普通话与英语流式识别。
- 逐节背诵和连续背诵。
- 实时识别文字、计时、暂停、提示、结束和重新背诵。
- 中文与英文的确定性经文对齐和可解释错误分类。
- “开始背诵”和“加入计划”入口。
- 四个内置计划模板以及自定义章节范围和 1–365 天。
- 本地计划、任务、尝试、掌握度、复习队列和统计持久化。
- 今日任务与学习统计的真实本地数据展示。

### 2.2 本轮不包含

- 云端识别、账号同步或服务器分析。
- 社交、小组、排行榜或共享计划。
- iOS、Windows 和 macOS 的模型二进制集成；领域接口和纯 Dart 判定逻辑必须保持可移植。
- 后台持续监听。

## 3. 技术方案

### 3.1 离线识别

采用 Sherpa-ONNX 双语量化流式模型。Android 原生层负责麦克风 PCM16 采集、重采样、模型生命周期和流式解码，Flutter 通过明确的平台接口接收部分文本、最终文本、音频状态和结构化故障。

选择理由：Sherpa-ONNX 提供 Android 实时流式识别和 Flutter 示例，支持中英文双语模型，并可在后续平台复用识别抽象。Vosk 的移动端模型更小，但公开的小型中文模型准确率不足以承担逐字经文核对；Whisper.cpp 的多语言模型在目标体积内更难兼顾流式延迟和耗电。

参考：

- https://k2-fsa.github.io/sherpa/onnx/android/build-sherpa-onnx.html
- https://k2-fsa.github.io/sherpa/onnx/flutter/asr-android.html
- https://alphacephei.com/vosk/models

模型与原生库全部随 APK 分发，不在运行时下载。构建产物目标为 180–300 MB。若压缩后超过 300 MB，先优化量化模型、无用文件和 ABI，不删除普通话或英语能力。

### 3.2 跨平台边界

领域层定义 `OfflineSpeechRecognizer`、`AudioSession`、`RecognitionEvent` 和 `RecognitionFailure`。Android 适配器是本轮唯一实现；UI、判定器、计划生成器和用户数据库不得直接依赖 Kotlin 或 Sherpa-ONNX 类型。

### 3.3 中文经卷名称

运行时读取规范化的 66 卷元数据，以 OSIS ID 查找 `zh-Hans`、`zh-Hant` 和 `en` 名称。经卷网格、已选经卷标题、章节标题、经文页标题、计划任务和统计都使用同一解析器。数据库和路由仍保存 OSIS ID，避免语言切换破坏稳定键。

## 4. 背诵流程

### 4.1 入口

经文页固定展示“开始背诵”和“加入计划”。“开始背诵”弹出模式选择：

- 逐节：每节独立识别和判定，通过后进入下一节。
- 连续：一次背诵当前选择范围，结束后整体对齐再投影回经节。

### 4.2 会话

开始后默认隐藏正文。页面显示中文经文范围、离线状态、录音状态、时长、实时识别文本、当前进度和操作按钮。用户可暂停、继续、请求首字提示、请求下一词提示、临时显示全文或结束。

显示全文会结束当前“无提示”尝试；提示次数进入熟练度计算。录音中断、模型错误或权限问题不得被记为用户背诵错误。

### 4.3 判定

判定器保留并显示权威经文原文，只对临时比较副本规范化：

- 中文忽略标点、空白和显示用简繁差异，统一常见数字读法。
- 英文忽略标点、大小写和不影响正文的缩写差异。
- 引用提示、章号和节号不参与正文评分。

对目标文本与最终识别序列做单调对齐，产生正确、插入、遗漏、替换和乱序。局部窗口内完整重说正确内容标记为“已自我纠正”。无法可靠判断的低可信片段进入“请重说”，不得自动判对或判错。

结果分为“准确通过”“通过但需复习”“需要重练”。结果页逐节展示证据，并提供重练错误、完整重背和完成。

## 5. 计划与进度

### 5.1 预置计划

- 7 天背诵诗篇 23 篇。
- 14 天背诵马太福音 5–7 章。
- 21 天背诵约翰福音 1–3 章。
- 30 天背诵腓立比书全书。

用户也可从经文页选择章节范围，输入 1–365 天。计划生成器按经文字数平衡每日新背任务，不简单平均章节数；同一经节不能在同一计划日重复。

### 5.2 今日与复习

今日页合并当日新内容、到期复习和之前未通过的重练项，不重复展示同一任务。首次通过后采用 1、3、7、14、30 天复习阶梯；失败内容进入次日重练。漏做任务保持未完成，不自动记为通过。

### 5.3 统计

统计页展示背诵次数、通过率、连续天数、已掌握经节、到期复习和最近趋势。所有指标从本地尝试与掌握度计算，不接入远程分析 SDK。

## 6. 本地数据

独立 `user.sqlite` 保存：

- 计划与计划状态。
- 每日任务和经文范围。
- 背诵尝试、模式、时间、结果和提示次数。
- 经节掌握度和复习队列。
- 模型版本、经文包 ID 和语义内容哈希引用。

经文库保持只读。替换经文来源或模型不得覆盖用户数据，也不得让不同文本修订静默继承掌握度。

原始录音默认只存在于会话缓冲或临时文件，判定完成后删除。没有用户主动选择时不得持久化录音。

## 7. 权限与错误

- 麦克风被拒绝：浏览、计划和统计继续工作；背诵页说明原因并允许重新申请。
- 模型初始化失败：显示可恢复错误，不创建失败尝试。
- 内存不足：释放模型和录音资源，保留计划任务状态。
- 录音中断：允许重新开始当前段落，不计内容错误。
- 用户主动离开：提示是否放弃当前未完成会话。
- 数据库写入失败：结果不得显示为已保存，允许重试。

## 8. 测试与验收

### 8.1 自动化测试

- 中文、繁体和英文经卷名解析。
- 中文/英文规范化、遗漏、替换、乱序、自我纠正和待确认。
- 计划按经文字数拆分、天数边界、无重复和复习日期。
- 开始背诵、模式选择、权限拒绝、提示、结束和结果页面。
- 加入计划、今日任务、统计更新和应用重启后的持久化。
- Android 原生模型初始化、PCM 输入和流式识别事件契约。

### 8.2 Android 验收

- 完全断网安装并首次启动。
- 中文目录显示中文经卷名和章节标题。
- 普通话和英语各完成一次逐节与连续背诵。
- 自动判定展示可解释差异。
- 创建一个预置计划和一个自定义天数计划。
- 关闭并重开应用后计划、任务、结果和统计仍存在。
- release APK 使用 v2 或更高签名验证通过，arm64 产物为 180–300 MB，并提供 SHA-256。

## 9. 完成定义

本轮完成要求：在支持 arm64、Android API 24 及以上的手机上，用户无需联网即可用中文目录选择经文，开始普通话或英语背诵，获得可解释判定，将章节加入预置或自定义天数计划，并在今日和统计页面看到重启后仍保留的真实数据。
