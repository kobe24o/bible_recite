# “我的”页面、艾宾浩斯复习与手机麦克风设计

## 目标

将现有“统计”入口升级为“我的”，在保留统计、成就和最近背诵的基础上，提供可选的艾宾浩斯复习功能。功能开启期间，达到用户正确率阈值的章节按固定遗忘曲线进入复习队列。录音输入始终使用手机内置麦克风，不因有线或蓝牙耳机连接而改变。

## 导航与页面

- 底部导航及宽屏导航栏的“统计”改为“我的”，图标改为个人账户语义。
- 路由可继续使用 `/statistics`，避免无必要的数据或深链迁移。
- 页面标题改为“我的”。
- 页面自上而下排列：艾宾浩斯设置、背诵统计、成就、最近背诵。
- 即使没有背诵结果，也必须显示艾宾浩斯设置；空状态仅用于统计内容区域，不能遮挡设置。

## 艾宾浩斯设置

- 开关默认关闭。
- 正确率阈值默认 `80%`，允许设置 `50%` 至 `100%`，按整数百分比保存。
- 设置卡片展示固定复习间隔：通过后的第 `1、2、4、7、15、30` 天。
- 设置完全保存在本地 SQLite，不依赖网络。
- 开启时记录 `enabled_at`；关闭时记录关闭状态。再次开启时更新 `enabled_at`，不补算关闭期间或重新开启之前的背诵。

## 进入复习队列的条件

- 只处理设置开启后完成的背诵结果。
- 背诵正确率 `>=` 当前阈值视为通过。
- 初次通过后，以通过日期为基准生成该章节第 1、2、4、7、15、30 天的复习节点。
- 阈值判定适用于逐节和连续背诵。
- 自动复习的单位是章节；即使本次只背诵了章内部分经节，也以该章节作为复习任务范围，任务打开完整章节。
- 同一译本、书卷、章节、计划日期和复习轮次只允许存在一个任务。

## 复习结果与重排

- 今日到期或已逾期且未完成的艾宾浩斯任务显示在“今日任务”。
- 任务可以点击进入对应完整章节，用户从章节页开始背诵。
- 达到阈值后，该复习节点完成；后续节点继续保持原计划日期。
- 未达到阈值时，以失败日期为新的基准，取消该章节尚未完成的后续节点，并重新生成第 1、2、4、7、15、30 天节点。
- 同一天对同一章节多次背诵时，以最新一次结果驱动状态；数据库唯一约束和事务保证不会重复排期。
- 关闭功能后，自动任务不再显示或推进，但历史任务与复习记录保留。再次开启只响应新 `enabled_at` 之后的背诵，不恢复旧队列。

## 数据模型

新增单行设置表：

- `ebbinghaus_settings`: `id=1`、`enabled`、`pass_threshold`、`enabled_at`、`updated_at`。

新增复习周期与节点表：

- `ebbinghaus_cycle`: 译本、书卷、章节、基准日期、当前状态、创建时间。
- `ebbinghaus_review`: 周期 ID、间隔天数、到期日期、完成状态、关联背诵结果、创建时间。
- 对周期、间隔和到期日期设置唯一约束，避免重复任务。

不把自动复习伪装成普通 `memorization_plan`，从而避免用户编辑或删除普通计划时破坏复习曲线。“今日任务”使用一个统一的展示模型合并普通计划任务和艾宾浩斯任务。

## 背诵结果关联

- 保存 `recitation_result` 后，在同一仓库调用艾宾浩斯调度器。
- 调度器读取当前设置和 `enabled_at`，以保存后的结果 ID、章节和正确率进行幂等处理。
- 从艾宾浩斯任务进入章节时传递 review ID；完成背诵后据此完成或重启对应周期。
- 用户从普通圣经页面主动背诵且通过，也可以创建新的章节复习周期。

## 录音输入

- 枚举输入设备后优先选择 `InputDeviceType.builtIn`。
- Android `manageBluetooth` 固定为 `false`，音频管理模式使用 `modeNormal`。
- 不再启动蓝牙 SCO，也不再执行蓝牙静音检测和自动回退。
- 如果系统没有返回内置设备条目，则使用默认设备但仍禁用应用主动蓝牙管理，并在界面显示“手机麦克风”。
- 蓝牙耳机只影响系统播放输出，不作为本应用录音输入。

## 错误处理

- 调度失败不能丢失已经保存的背诵结果；页面显示本地排期失败提示，可在下次打开“我的”或“今日”时幂等重建。
- 数据迁移使用 `CREATE TABLE IF NOT EXISTS` 和明确版本升级，保留现有计划、统计和成就数据。
- 非法阈值在仓库层限制到 `50%–100%`，界面层同样限制可选范围。

## 测试与验收

- 领域测试覆盖阈值边界：`79%` 不通过、`80%` 通过。
- 仓库测试覆盖默认设置、开关、开启时间、不补算旧结果、六个节点、去重、失败重排、关闭后隐藏。
- 页面测试覆盖“统计”改为“我的”、无数据时仍显示设置、阈值编辑和今日任务点击。
- 录音路由测试覆盖连接蓝牙设备时仍选择内置麦克风，且录音配置禁用蓝牙管理。
- 完整 Flutter 测试、静态分析和 release APK 构建必须通过。
- 发布版本从 `1.0.1+2` 升至 `1.0.2+3`，APK 文件名包含版本号。
