# 品牌、标点恢复、成就与蓝牙录音设计

## 目标

在不改变离线识别和经文来源接口的前提下，为圣经背诵 App 增加统一品牌 Logo、带原文标点的彩色背诵结果、鼓励型成就勋章，并修复连接蓝牙耳机后录音无有效输入的问题。

## 范围与边界

本版本包含四个边界清晰的功能单元：品牌资源、背诵结果渲染、成就规则、Android 音频路由。它们共同进入一个新版 APK，但分别测试。经文内容继续来自本地权威经文包；标点不由模型生成；成就和录音结果不上传网络。

## 1. Logo 与品牌资源

采用已确认的 A 方案“经卷与书签”：展开的圣经为主体，右页带金色书签。主色为深森林绿，辅色为暖金色，经文页为象牙白。图标不包含文字，确保在手机桌面小尺寸下仍可辨识。

交付资源包括：

- 1024×1024 主 Logo PNG。
- Android 自适应图标前景与纯色背景。
- Android 各密度普通和圆形启动图标。
- iOS AppIcon 所需尺寸。
- 应用内可复用 Logo 和启动页图形。

启动页使用同一图形和配色，不再显示默认 Flutter 标识。图标边缘保留足够安全区，圆形或圆角方形裁切时不截断经卷和书签。

## 2. 保留原文标点的背诵结果

### 2.1 比对与展示分离

正确率计算继续使用标准化字符序列：忽略空格、标点和英文大小写，只比较汉字、字母和数字。展示层使用权威经文原文作为骨架，将比对得到的状态重新映射回原文位置。

### 2.2 显示规则

- 正确文字为绿色。
- 错字或多字为红色。
- 中间漏字显示红色 `_`。
- 顺序错误为橙色。
- 实时识别时尚未背到的文字为浅灰色。
- 结束背诵后，尚未背诵的文字转为红色。
- 逗号、句号、分号、冒号、引号等使用普通正文色，永远不参与正确率，也不显示错误状态。
- 英文保留经文原有大小写、空格和标点，但这些格式字符不参与正确率。

为了稳定映射，标准化阶段同时保存每个可比较字符在原文中的索引。对齐器只处理标准化字符；渲染器按索引合并原文标点和对齐状态。识别产生的额外文字没有原文位置时，紧邻当前对齐点以红色显示。

## 3. 鼓励型成就勋章

### 3.1 数据模型

新增成就定义目录和 `achievement_unlock` 本地表。解锁记录包含成就 ID、获得时间和触发来源。成就定义与规则由应用版本管理，解锁记录持久保存。规则检查在背诵结果保存、计划创建和计划任务完成后执行。

同一成就只解锁一次。一次操作触发多个成就时按规则优先级依次展示。已有历史数据在首次打开新版统计页时补算，满足条件的成就写入解锁表，但补算时集中显示一张“获得多个成就”卡片，避免连续弹窗干扰。

### 3.2 首版规则

| 成就 | 条件 |
| --- | --- |
| 初次开口 | 完成第 1 次背诵 |
| 小小坚持 | 累计背诵 3 次 |
| 渐入佳境 | 累计背诵 10 次 |
| 熟能生巧 | 累计背诵 25 次 |
| 背诵达人 | 累计背诵 50 次 |
| 经文百炼 | 累计背诵 100 次 |
| 三日同行 | 连续背诵 3 天 |
| 一周坚持 | 连续背诵 7 天 |
| 月度同行 | 连续背诵 30 天 |
| 十节种子 | 累计完成 10 节 |
| 五十节成长 | 累计完成 50 节 |
| 百节丰收 | 累计完成 100 节 |
| 初次优秀 | 单次正确率不低于 80% |
| 精准背诵 | 单次正确率不低于 90% |
| 一字不差 | 单次正确率 100%，且有效经文不少于 20 个字符 |
| 完成一章 | 同一章的全部经节都至少完成背诵一次 |
| 计划启程 | 创建第一个背诵计划 |
| 计划完成 | 第一次完成整个计划 |

“连续天数”按本地自然日计算；同一天多次背诵只计一天。累计节数按保存结果覆盖的经节数累加。完成一章使用本地经文仓库的实际经节范围核验，省略节位不要求背诵。

### 3.3 交互

新成就解锁后显示金色庆祝卡片并触发轻微震动。卡片包含勋章图形、名称、条件和获得时间。统计页增加“我的成就”网格：已解锁为森林绿和暖金色，未解锁为灰色；未解锁项显示下一目标和当前进度。

## 4. 蓝牙耳机录音修复

### 4.1 已确认根因

当前 Android 清单只有 `RECORD_AUDIO`。所用 `record_android 2.1.2` 组件默认管理蓝牙 SCO，但其文档明确要求应用添加 `MODIFY_AUDIO_SETTINGS`。用户现象为“界面进入录音状态但没有识别文字”，与 SCO 通道未成功建立、录音流无有效蓝牙输入一致。

### 4.2 路由策略

- Android 清单添加 `MODIFY_AUDIO_SETTINGS`。
- 开始录音前枚举输入设备；连接可通话蓝牙耳机时优先选择 Bluetooth SCO 输入。
- 蓝牙设备只有 A2DP 输出、没有 SCO 麦克风时使用手机麦克风。
- Android 12 及以上使用系统通信设备路由；旧版 Android 使用兼容 SCO 路由。
- 页面显示当前输入来源：“蓝牙耳机麦克风”或“手机麦克风”。
- 建立蓝牙路由后监测 PCM 输入。持续两秒无有效音频时停止当前流，以手机麦克风重新开始，并提示“蓝牙麦克风无声音，已切换到手机麦克风”。
- 停止、退出或异常结束时释放录音器和通信设备路由，避免影响音乐或通话应用。

自动降级只发生一次，防止设备异常时循环重启录音。若手机麦克风也无有效音频，则停止录音并显示可操作错误信息。

## 5. 错误处理与兼容

- Logo 资源生成失败不影响现有图标，只有验证后的完整资源集才替换当前图标。
- 标点映射发生异常时回退到当前无标点彩色结果，不影响统计保存。
- 成就规则检查失败不阻断背诵结果或计划保存；错误记录在本地日志，下次进入统计页重试。
- 数据库使用增量迁移，仅新增成就表，不修改现有背诵结果含义。
- 蓝牙路由失败时自动降级手机麦克风，不要求用户断开耳机。

## 6. 测试与验收

### 自动测试

- 标点、空格和大小写不影响正确率。
- 中文及英文原文标点能在正确位置恢复。
- 正确、错误、漏字、错序、未完成状态与原文标点可同时渲染。
- 每条成就规则的边界值、重复解锁、连续自然日和历史补算。
- 蓝牙 SCO 优先、A2DP 回退、无音频一次性降级及路由释放。
- Android 合并清单包含 `RECORD_AUDIO` 和 `MODIFY_AUDIO_SETTINGS`。

### 人工验收

- Android 桌面、圆形启动器和启动页检查 Logo 裁切及清晰度。
- 使用中文和英文经文完成一次实时背诵，确认标点舒适且不影响正确率。
- 触发至少两个勋章，确认即时提示、震动、统计页状态和重复进入不再弹出。
- 使用真实蓝牙通话耳机连接后开始背诵，确认显示蓝牙麦克风、产生实时文字；再使用仅支持输出的设备验证手机麦克风回退。
- 运行全部 Flutter 测试、静态检查并构建 release APK，记录文件大小和 SHA-256。
